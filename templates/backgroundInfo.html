Background Information Page


Specific technologies needed:
With UI: docker and python/js
Without: just python/js probably.

There is ultimately 2 ways to do this, one way with a UI and one way without. Doing it without should be fairly simple but we would need to make the program large enough to have it not be blatantly obvious.
Without UI scenario:
Give the user a program in which they can explore. it needs to have a decent amount of meat on it as to hide the custom package. This would imply that the program is written either python or JS, either one is fine since there isn't a UI. The user would then look through the package and attempt to identify which ones are out of date using a npm command or a pip command.
Upon finding the custom package (or packages) the user would then be directed to find the package with the security issue online(https://cve.mitre.org/cve/search_cve_list.html). They would then have to find the cve and either exploit it to obtain the flag, or they can update it and the program can throw an error to release the flag.
Codelab steps:
download the program to run/download repository
let the user use the application for a brief moment to understand what it does
inform the user of the command needed to find the age of packages
ask the user to search the packages for ones with CVEs
ensure the user knows the correct package by asking for it in the code lab
Ask the user to attempt to update it
Let the user rerun the program and either watch it break or have it throw the flag as an error
